# FIRE — AI-Stock Research Dashboard

A local dashboard for **long-term** investing in the AI value chain.
Pulls prices, fundamentals, earnings, SEC filings, and social
sentiment for a watchlist you control. Asks your **own Claude
subscription** to write a research thesis on each ticker, tailored to
the investing philosophy you put in `STRATEGY.md`.

Everything runs on your machine. No server, no auth, no cloud, no
data leaves your laptop.

> Informational only. Nothing here is investment advice. Valuation
> labels and Claude-authored theses are research starting points, not
> buy/sell signals.

---

## Prerequisites

1. **Python 3.10 or newer**
2. **The `claude` CLI** — install from
   [docs.claude.com/en/docs/claude-code](https://docs.claude.com/en/docs/claude-code)
   and `claude login` so it's ready to use.
3. **An active Claude subscription** (Pro / Max / Team) — every thesis
   call uses *your* subscription. The dashboard never bills you;
   Anthropic does.

The dashboard still renders without the `claude` CLI on `PATH` — you
just see a heuristic mock thesis instead of the strategy-tailored one.

## Install

```bash
git clone <this repo> fire
cd fire
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

## First run

```bash
streamlit run app.py
```

The repo ships with a sensible default `STRATEGY.md` and `watchlist.yaml`.
Open the app and:

1. Visit the **STRATEGY** tab → edit your horizon, return target,
   themes, and hard rules. Save.
2. Visit the **WATCHLIST** tab → add the tickers you actually care
   about. The default list is AI-value-chain heavy; swap in your own.
3. Hop to **RESEARCH** → pick a ticker → click "◇ Generate Claude
   thesis". First call takes ~60s and uses your Claude subscription.

> 💡 **Editing `STRATEGY.md` or `watchlist.yaml` directly:** both are
> tracked files. If you pull upstream updates later, you may hit merge
> conflicts on them — `git stash` your local edits before pulling, or
> fork the repo if you expect to diverge heavily.

## Daily use

```bash
# Refresh snapshots + sentiment for the whole watchlist (~30s)
python -m fire.collector

# Same, plus run Claude thesis on every `holding` ticker so the
# dashboard is instant when you open it (free if cached)
python -m fire.collector --pre-warm

# Launch the dashboard
streamlit run app.py
```

## How the strategy file works

`STRATEGY.md` is YAML frontmatter + Markdown body:

```markdown
---
horizon_years: [2, 3]
target_return_multiple: 10
position_style: concentrated
themes:
  - AI super-cycle infrastructure
hard_rules:
  - No short-term trading
---

# My philosophy

Free-form prose explaining what you look for, what you avoid, and
why. Claude reads the whole document as system context for every
thesis card.
```

The Strategy tab in the UI renders the frontmatter as form fields and
the body as a textarea — you don't have to touch the file by hand.

The cache key for every Claude call includes a hash of this file, so
editing your strategy automatically invalidates the right cache
entries. Old theses stay around (you can scroll back); new calls use
the new lens.

## What gets created locally

- `data/fire.db` — SQLite database, created on first launch. Holds
  snapshots, earnings, filings, sentiment, and Claude response cache.
- `briefs/` — Markdown briefs generated by `python -m fire.brief`.
- `.env` (optional) — copy from `.env.example` if you want to set a
  Reddit user-agent or an X bearer token. Both are optional; the
  core dashboard runs without any keys.

To reset price/sentiment history:
```bash
rm data/fire.db
```

## Notes & limits

- **yfinance is an unofficial data source.** Occasionally a ticker
  returns partial data. The collector skips failures rather than
  crashing and reports which tickers had no price.
- **Cohort percentile bars in the mockups were illustrative.** The
  current implementation colors KPI tags ("cheap" / "rich" /
  "elite"), not full percentile bars. Real peer cohorts are tracked
  in `peer_percentiles` and computed when populated.
- **No synthetic data.** If a source doesn't have a number, the
  section is left empty rather than filled with made-up values.
- **This tool does not place trades or move money.**
